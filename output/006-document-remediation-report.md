## 🔍 关键修改说明

### 为什么不能实现自动化调用？

**OpenClaw 版本限制**：
1. 当前版本（2026.3.2）不支持 subagent 功能
2. 配置 `subagents.allow` 无效
3. 语法 `@agent-id` 不被识别

**实际测试结果**：
```bash
# 尝试配置 subagents.allow
{
  "subagents": {
    "allow": ["ceo-bezos", "cto-vogels"]
  }
}

# Gateway 启动时报错
# "Unrecognized key: allow"
```

**可行的替代方案**：
- 指挥官使用**角色切换**模式
- 通过**文档共享**实现协作
- 用户**手动触发**不同 Agent

### 文档定位调整

**原定位**：完全自动化的多 Agent 团队
**新定位**：文档驱动的协作模式

**核心差异**：
```
原始设计：用户 → 指挥官 → 自动调用 Agent → 结果
实际实现：用户 → 指挥官（切换角色） → 文档协作 → 结果
```
